# Prompt Stitch.ai - Desain UI/UX DisaCare Bandung

Berikut adalah prompt komprehensif yang dirancang khusus untuk membuat mockups UI/UX premium yang ramah disabilitas di **stitch.ai**. Anda dapat menyalin teks di bawah ini ke dalam stitch.ai untuk men-generate antarmuka.

---

### Teks Prompt untuk Stitch.ai (Salin teks di bawah ini)

```text
Create a high-fidelity, premium, responsive web application UI/UX design for "DisaCare Bandung" - an inclusive spatial accessibility portal for disabled people in Bandung, Indonesia. 

The application must adhere to WCAG AAA accessibility guidelines (excellent readability, high contrast, clean structure) while looking modern, visually stunning, and highly professional, utilizing Tailwind CSS style tokens.

Brand Color Palette:
- Primary: Royal Blue (#004ac6) for safety and trust
- Success: Green (#007f36) for high accessibility
- Warning: Amber/Yellow (#da9100) for moderate accessibility
- Error/Alert: Crimson Red (#ba1a1a) for low accessibility
- Dark Mode / High Contrast Toggle: Pure black background (#000000) and bright yellow text (#FFD700)
- Fonts: Atkinson Hyperlegible Next or Inter (clean sans-serif tailored for low-vision readability)

Screens & Layouts to Generate:

1. Landing Page (Halaman Muka - Public Splash Screen):
- A warm, premium, and clean welcome screen before entering the main map directory.
- Features a large hero section with the title: "DisaCare Bandung" and subtitle: "Portal Informasi Aksesibilitas Spasial Ramah Disabilitas Kota Bandung".
- Features "Didi" the animated mascot: a friendly, cute, animated round helper robot mascot with big digital eyes and expressing warmth. Didi is waving at the user and wearing accessibility gears (like headphones representing screen reader, or holding a magnifying glass representing vision aid).
- Prominent CTA buttons: "Cari Fasilitas Publik" (Primary Royal Blue button, leads to directory) and "Masuk Sebagai Kontributor" (Outline button, leads to auth).
- A section showcasing the three primary accessibility levels with color-coded badges (Green: Sangat Aksesibel, Yellow: Cukup Aksesibel, Red: Perlu Perbaikan) with Didi explaining each level.

2. Main Map Directory & Search Dashboard (Public View):
- A split-screen layout: left side is a searchable sidebar with category filters (Mall, Kampus, Rumah Sakit, Kantor Pemerintah, Taman, Stasiun, Lainnya), and the right side is an interactive map interface (OpenStreetMap style) with custom pins.
- Place cards in the sidebar: each card shows the facility name, address, category, a photo proof, and a large accessibility score badge (e.g., "85% Sangat Aksesibel").
- Didi the mascot appears on the map markers and cards, holding small flags representing accessibility scores (e.g. Green, Yellow, Red flags).
- Persistent Accessibility Widget (Sticky Bottom Bar): clean controls for Text-to-Speech (TTS), High Contrast (Black-Yellow Mode), and Font Size Resizer (Font+ and Font-).

3. Place Detail Page (Public View):
- A detailed breakdown of a selected place. Displays the name, category, official vs user-contributed badge, and address.
- Left column: Large photo gallery showcasing physical proofs (like ramp photos, disability toilets).
- Right column: Mini map displaying the precise location, and a large circle score meter (e.g. 83%).
- A grid of accessibility checklist with icons: Ramp Kursi Roda (Wheelchair Ramp), Toilet Disabilitas (Accessible Toilet), Guiding Block, Parkir Khusus (Reserved Parking), Pintu Lebar/Otomatis (Wide/Automatic Door), Lift/Elevator. Checked items show a green tick, unchecked show a red cross.
- Mascot Didi is shown on the sidebar holding a clipboard with the checklist, pointing happily to the green checks.
- Prominent "Dengarkan Informasi (Text-to-Speech)" speaker button next to the name.

4. Contributor Login & Portal (Protected View):
- A sleek, split-screen login page with a clean form on one side. On the other side, a premium illustration of mascot Didi holding a key with Bandung landmarks in the background.
- Registration page containing Name, Email, Password, and a dropdown role ("Kontributor").
- "Laporkan Fasilitas Baru" Form Screen: Multi-step clean input form. 
  * Step 1: Basic Information (Name, Category dropdown, Address, Coordinates map pin selector).
  * Step 2: Facility Checklist (Interactive switches/checkboxes).
  * Step 3: Physical Proof Upload (Drag-and-drop file upload zone for photos).
  * Mascot Didi is shown in a builder hat holding a camera, showing a preview of the photo proof.

5. Admin Moderation Dashboard (Protected View):
- Clean list of pending verification items showing Place Name, Contributor Name, date, and a "Review" button.
- Verification page: side-by-side comparison of the uploaded data checklist and the physical photo proof. 
- Clear "Approve" (Green button) and "Reject" (Red button) moderation controls.
- Mascot Didi shown wearing a detective glass, magnifying the photo proof to emphasize the verification process.
```
